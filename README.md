# מערכת ניטור תשדירי רדיו – קול רגע

זו מערכת מלאה ופשוטה יחסית להפעלה:

- מעלים קבצי תשדירים
- המערכת מאזינה לשידור החי של קול רגע
- מזהה מתי אחד התשדירים שודר
- שומרת תאריך, שעה, ודאות וקטע הוכחה
- שולחת דוח יומי למייל
- אפשר לשנות קבצים ומייל יעד מתי שרוצים

## איך המערכת בנויה

יש כאן 2 חלקים:

1. **Web** – מסך הניהול בדפדפן
2. **Worker** – מנוע הרקע שמאזין לשידור 24/7

שניהם עולים מאותו קוד ב-Railway.

## מה צריך לפתוח

### 1) חשבון Supabase
צריך לפתוח פרויקט חדש ב-Supabase.

אחרי שפתחת:
- תעתיק את `Project URL`
- תעתיק את `service_role key`
- תעתיק את פרטי ה-Postgres connection string

### 2) חשבון Resend
צריך:
- לפתוח חשבון
- לאמת דומיין למשל `reports.yourdomain.com` או הדומיין הראשי שלך
- ליצור API Key
- להגדיר כתובת שולחת, למשל `reports@yourdomain.com`

### 3) חשבון Railway
צריך לפתוח פרויקט חדש ולחבר אליו את ה-GitHub repo של הקוד הזה.

## הגדרה ב-Supabase

### A. צור 2 buckets
ב-Storage תיצור:
- `ad-files`
- `evidence-files`

### B. צור טבלאות
ב-SQL Editor של Supabase, תריץ את הקובץ:
- `sql/init.sql`

## הגדרה ב-Railway

ב-Railway תעלה את הקוד הזה ל-GitHub ואז:

### שירות 1 – web
- בחר deploy מה-repo
- Start command:
  - `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

### שירות 2 – worker
- מאותו repo תיצור עוד service
- Start command:
  - `python -m app.worker`

### Environment Variables
תכניס בכל אחד מהשירותים את כל הערכים מקובץ `.env.example`

## איך משתמשים במערכת

### העלאת תשדיר
במסך `תשדירים`:
- שם התשדיר
- תיאור אופציונלי
- העלאת MP3/WAV

### שינוי מייל לדוחות
במסך `הגדרות`:
- אפשר להחליף מייל יעד

### צפייה בזיהויים
במסך `זיהויים`:
- רואים מתי כל תשדיר שודר
- אפשר לשמוע קטע הוכחה

## הערות חשובות

1. מומלץ להעלות **קובץ מקורי** של התשדיר
2. אם התשדיר עבר עריכה קשה מאוד בשידור, ייתכן שצריך לכוון סף
3. מומלץ להתחיל עם 2–5 תשדירים ורק אחר כך להרחיב
4. המערכת בנויה ל-MVP אמיתי ופשוט, לא למערכת ענק

## קבצים חשובים

- `sql/init.sql` – בסיס הנתונים
- `app/main.py` – אפליקציית ה-web
- `app/worker.py` – מנוע הניטור
- `app/services/audio_matcher.py` – מנגנון הזיהוי
- `app/templates/` – מסכי המערכת
